Contexto
Abertura

Apresente quem você é, o que é a Levante e por que o projeto existe. Sem entrar em tecnologia ainda.

"A Levante é uma operação de marketing especializada no setor automotivo. A gente cuida de tráfego pago, criativo e automação para empresas que vendem para revendas. Construímos nosso próprio CRM do zero, com IA integrada — ele é um produto que entregamos para os nossos clientes usarem no dia a dia comercial deles, e também usamos internamente. Ele já funciona bem — mas falta uma peça essencial: os usuários do CRM precisam conversar com os clientes deles pelo WhatsApp direto dentro do sistema, sem ficar trocando de tela. É isso que você vai construir."
Não precisa falar de tecnologia aqui. Só contexto de negócio. Ela vai entender o "porquê" antes do "como".

Objetivo: ela entender onde o WhatsApp vai se encaixar antes de ver a spec. Siga essa ordem.

  1. Pipeline (Boards) — Abre a tela do quadro Kanban. Mostra os cards. Fale: "Cada cartão é um cliente que a gente está trabalhando."

  3. Botão de WhatsApp atual — Mostra o botão que já existe no cockpit (abre wa.me). Fale: "Hoje clica aqui e abre o WhatsApp do celular. Queremos que a conversa apareça aqui dentro, sem sair do sistema."
  4. Configurações — Abre a tela de settings. Fale: "Vai ter uma seção nova aqui para o cliente conectar o número de WhatsApp dele."

A Documentação
A spec do projeto

Abre o HTML da spec. Não leia tudo com ela — só mostre o mapa. Ela vai ler sozinha depois.

  1. Abre na seção "Por onde começar" (já aparece destacada no menu). Fale: "Aqui tem a ordem que você lê o código antes de começar a escrever qualquer coisa."
  2. Vai para a seção "Implementação". Mostra os 4 cards de fase. Fale: "Dividimos em 4 fases. Cada fase entrega algo que o usuário já consegue usar. Você vai ler os detalhes com calma." — Não precisa explicar o técnico de cada fase.
  3. Mostra a seção "Alinhamento com o CRM". Explica: "Verde = já existe no código, você só usa. Amarelo = precisa de atenção. Vermelho = você vai criar do zero."
  4. Mostra a seção "Padrões do Código". Fale: "Aqui tem exemplos reais do código do sistema pra você copiar o padrão. Não precisa inventar do zero."
"Essa documentação foi construída junto com uma análise do código real do sistema. Não é só teoria — tem código concreto pra você usar. Mas não precisa ler agora. Quero só que você veja o tamanho do projeto e me diga se faz sentido pra você."
Se ela começar a fazer perguntas técnicas agora — ótimo sinal. Responda o que souber, e o que não souber fala: "Isso tá explicado na documentação, mas posso te conectar com quem construiu o sistema se precisar."

10min
Conhecer ela
As 4 perguntas

Faça essas 4 perguntas e escuta mais do que fala. Não precisa entender a resposta técnica — presta atenção na confiança dela.

Pergunta 1
"Você já usou TypeScript? Com que frequência?"
TypeScript é a linguagem do sistema. Se ela usa com frequência, vai entrar mais fácil.
Pergunta 2
"Já integrou alguma API externa num sistema existente? Como foi?"
O WhatsApp API é o núcleo do projeto. Se ela já fez algo parecido, é uma boa base.
Pergunta 3
"Já trabalhou com Supabase ou banco de dados com login e multi-usuário?"
O CRM usa Supabase com isolamento por empresa. Não precisa ter feito igual — só parecido.
★ Mais importante
"Quando você trava num problema que nunca viu antes, o que você faz?"
Resposta esperada: "Vou na documentação, pesquiso, peço ajuda quando trava de vez." Se ela travar na própria pergunta — atenção.
Você não precisa avaliar a resposta técnica dela — você não é dev. Avalia a postura: ela fala com segurança? Ela dá exemplos reais? Ela demonstra curiosidade? Isso vale mais do que saber as respostas certas.

5min
Fechar
Próximos passos

Dependendo de como foi o Bloco 4, siga um dos dois caminhos:

✓ A conversa foi bem
Propõe só a Fase 1 como teste. Deixa ela falar o prazo e o valor primeiro — nunca sugira número antes dela.
"Antes de a gente fechar o projeto inteiro, queria começar pela Fase 1 — banco de dados e o sistema recebendo mensagem. Qual prazo você precisaria pra isso, e qual valor faria sentido pra você?"
✕ Ela hesitou muito
Não fecha na hora. Dá um recuo respeitoso. Nada de pressão — você vai sair no prejuízo se contratar na dúvida.
"Adorei conversar. Vou revisar umas coisas aqui e te retorno com os próximos passos, tudo bem?"

Se ela apresentou prazo e valor razoáveis, feche assim:

"Então combinado: Fase 1 entregue até [data que ela falou]. Quando eu validar que tá funcionando, a gente acerta o pagamento e parte pra Fase 2. Vou te mandar a documentação no email hoje."
Referência de prazo aceitável para a Fase 1: até 6 semanas. Se ela pedir mais, não é sinal ruim — é sinal de que ela entendeu a complexidade.
